PHP Popup Modal System - Documentation

This documentation provides a comprehensive guide for setting up, configuring, and deploying the PHP Popup Modal System. This system utilizes MongoDB Atlas for database storage, offering flexibility and scalability.

1. Project Overview & Quick Start

The PHP Popup Modal System is a customizable web application designed to display popup modals based on various triggers such as delay, scroll percentage, or exit intent. It includes an administrative panel for configuring popup settings, with data securely stored in a MongoDB database.

This plugin enables full control over popup display via an administrative interface.

Key Technologies Utilized in This Project:

  • PHP 8.3+
  • Composer (PHP Dependency Manager)
  • MongoDB PHP Driver
  • MongoDB Atlas (Managed Cloud Database)
  • Render (Cloud Hosting Platform)

1.2. Quick Start Guide (Basic Local Usage)

For initial local setup of the plugin, follow these fundamental steps. Refer to subsequent sections for detailed configuration.

  1. Upload the Folder: Place the entire popup directory into your web server's document root (e.g., htdocs/, public_html/, or your PHP development server's root).
  2. Install PHP Dependencies: Open your terminal within the popup directory and execute: 
    composer install
    Note: Ensure PHP 8.3+ and the php_mongodb.dll extension are installed and enabled (refer to Section 2.1) prior to running Composer.
  3. MongoDB Atlas Setup (Brief):
    • Create a free MongoDB Atlas M0 cluster (refer to Section 3.1).
    • Create a Database User (Section 3.2) and **securely record the generated password.**
    • Add your **current local IP address** to the Network Access whitelist (Section 3.3).
    • Obtain your MongoDB Atlas connection string (Section 3.4) and replace the <username> and <password> placeholders with your actual credentials.
  4. Configure Database Connection: Open config/config.php.
    • Locate the $mongoUri variable. For local testing, **uncomment the hardcoded string** and insert your full MongoDB Atlas connection string (with actual username and password).
    • Modify 'your_demo_database_name' to your preferred database name.
  5. Access the Admin Panel: Navigate to your local admin panel URL in your web browser (e.g., http://localhost/popup/admin/ or http://localhost:8000/admin/).
  6. Configure Popup: Utilize the admin panel user interface to define popup content, triggers, and display settings. These settings will be persisted in MongoDB Atlas.
  7. Visit Frontend Demo: Access your frontend demonstration URL (e.g., http://localhost/popup/frontend/index.php or http://localhost:8000/frontend/index.php) to observe the popup functionality.

2. Local Development Setup (Detailed)

For a thorough setup of this system in your local development environment, follow these detailed instructions:

2.1. Prerequisites (Essential Tools)

  • PHP 8.3 or later: Ensure PHP is properly installed and configured on your operating system (e.g., through XAMPP, WAMP, or a direct PHP installation).
  • Composer: The standard dependency manager for PHP projects. Download it from: Composer Official Download.
  • MongoDB PHP Driver (ext-mongodb DLL): This is a compiled PHP extension required for MongoDB connectivity.
    1. Visit the PECL website for the MongoDB extension: pecl.php.net/package/mongodb.
    2. Download the latest stable 64-bit TS (Thread Safe) version of the mongodb extension that is compatible with your PHP 8.3 installation (e.g., a file named similar to php_mongodb-2.x.x-8.3-ts-vs16-x64.zip).
    3. Extract the downloaded ZIP file and copy php_mongodb.dll into your PHP's extensions directory (e.g., C:\php8\ext\).
    4. Open your php.ini configuration file (e.g., C:\php8\php.ini).
    5. Add or uncomment the following line in php.ini to enable the extension: extension=php_mongodb.dll
    6. Save the php.ini file and then restart your web server (Apache/Nginx) or PHP development server for the changes to take effect.
    7. Confirm the extension is loaded by reviewing your phpinfo() output (search for "mongodb").
  • Git: A version control system essential for cloning the project repository.

2.2. Obtain the Code (Cloning & Dependencies)

  1. Clone the repository to your local machine: 
    git clone [YOUR_REPOSITORY_URL]
  2. Navigate into the project's root directory: 
    cd popup
  3. Install PHP Composer dependencies: 
    composer install
    Note: If Composer reports errors regarding a missing ext-mongodb extension, ensure you have successfully completed the prerequisite steps in Section 2.1 to install the DLL and restarted your server.

3. MongoDB Atlas Database Setup (Free Tier)

This project utilizes MongoDB Atlas for its cloud-hosted database. A free (M0 Sandbox) cluster can be provisioned for development and testing purposes.

3.1. Create a MongoDB Atlas Account & Cluster

  1. Visit cloud.mongodb.com to sign up or log in.
  2. Create a new project (if applicable, to organize your clusters).
  3. On the "Build a Database" page, select the "Shared" option, corresponding to the M0 Free Tier.
  4. Choose a Cloud Provider and a Region that is geographically proximate to your intended Render application deployment for optimal performance (e.g., AWS / N. Virginia (us-east-1)).
  5. Assign a descriptive name to your cluster (e.g., popup-db-cluster).
  6. Click "Create Cluster" and await its provisioning, which typically takes 3-7 minutes.

3.2. Create a Database User

A dedicated database user must be created for your PHP application to establish a secure connection to the database.

  1. Within the Atlas dashboard, navigate to "Database Access" under the "Security" section.
  2. Click "+ Add New Database User".
  3. Select "Password" as the authentication method.
  4. Enter a Username (e.g., phpuser) and utilize the "Autogenerate Secure Password" feature.
    CRITICAL: Immediately copy and securely store this autogenerated password. It is essential for your application's database connection.
  5. For "Database User Privileges", select "Read and write to any database" for typical application access.
  6. Click "Add User".

3.3. Configure Network Access (IP Whitelisting)

You must configure MongoDB Atlas to allow inbound connections only from specified IP addresses.

  1. In the Atlas dashboard, navigate to "Network Access" under the "Security" section.
  2. Click "Add IP Address".
  3. For Local Development (Your Current IP):
    • Click "Add Current IP Address" to automatically detect and add your public IP.
    • Provide a brief description (e.g., "Development PC IP").
    • Click "Confirm".
  4. For Render Deployment: Render's outbound IP addresses will be added later (refer to Section 5.3).
  5. Click "Finish and Close".

3.4. Obtain the Connection String

Once your cluster's status indicates "Available", retrieve the necessary connection string for your application.

  1. In the Atlas dashboard, proceed to "Database Deployments" > "Clusters".
  2. Click the "Connect" button adjacent to your cluster.
  3. Select "Connect your application" from the dialog.
  4. For "Driver", choose "PHP". For "Version", select "1.11 + mongodb-1.10 or later".
  5. Copy the displayed connection string. It will be formatted as: 
    mongodb+srv://<username>:<password>@yourcluster.abcde.mongodb.net/?retryWrites=true&w=majority
  6. CRITICAL: In the copied string, replace <username> with the actual database username (e.g., phpuser) and <password> with the securely stored password from Section 3.2. Ensure no angle brackets (< >) remain.
  7. Save this complete and corrected connection string. It will be used in config.php and configured on Render.

4. Application Configuration (config.php)

The config.php file manages the database connection logic. It requires modification to integrate with MongoDB.

IMPORTANT: The $mongoUri variable in config.php must be adjusted based on whether you are developing locally or deploying to Render. 

<?php
require __DIR__ . '/../vendor/autoload.php'; // Essential for Composer-managed dependencies

// Base URL definitions for localhost and live environments
$base = (isset($_SERVER['HTTPS']) ? 'https://' : 'http://') . $_SERVER['HTTP_HOST'];
define('PLUGIN_ROOT', $base . '/');
define('PLUGIN_ASSETS', PLUGIN_ROOT . 'assets/');
define('PLUGIN_UPLOADS', PLUGIN_ROOT . 'uploads/images/popup/');

// MongoDB Atlas connection settings
// --- FOR LOCAL TESTING ONLY ---
// Paste your full MongoDB Atlas Connection String here (with actual password).
// COMMENT THIS LINE OUT BEFORE COMMITTING FOR RENDER DEPLOYMENT.
$mongoUri = 'mongodb+srv://[YOUR_DB_USERNAME]:[YOUR_DB_PASSWORD]@[YOUR_MONGODB_ATLAS_CLUSTER_HOSTNAME]/?retryWrites=true&w=majority';
// -----------------------------

// --- FOR RENDER DEPLOYMENT ---
// UNCOMMENT THIS LINE BEFORE COMMITTING FOR RENDER DEPLOYMENT.
// $mongoUri = getenv('MONGO_URI');
// -----------------------------

try {
    $mongoClient = new MongoDB\Client($mongoUri);
    // Select your desired database name. This database will be created upon the first write operation.
    $mongoDb = $mongoClient->selectDatabase('your_demo_database_name'); // <<< CHANGE THIS to your desired database name (e.g., 'popup_db')

    // Optional: Uncomment the line below for connection verification during local testing
    // echo "Successfully connected to MongoDB Atlas locally!";

} catch (MongoDB\Driver\Exception\Exception $e) {
    die("MongoDB Connection failed: " . $e->getMessage());
}
?>

Additionally, ensure your functions.php file is updated to utilize the $mongoDb global variable for all database operations (e.g., $mongoDb->selectCollection('popup_settings')->findOne()), replacing any previous database interaction logic.

5. Deployment to Render

5.1. Update .gitignore (Repository Cleanliness)

It is crucial to maintain a clean Git repository by preventing the commitment of unnecessary files. Ensure your .gitignore file, located in your project's root directory, includes the following: 

# Composer dependencies - These are generated and should not be version controlled
/vendor/

# Environment variables - Contains sensitive information; exclude from repository
.env

# IDE and OS specific files - Local configurations and temporary files
.vscode/
.DS_Store
Thumbs.db

# Log files - Dynamically generated and temporary
*.log

Commit this .gitignore file to your repository.

5.2. Configure config.php for Render Deployment

Prior to committing for Render deployment, adjust your config.php file as described in Section 4: uncomment the line $mongoUri = getenv('MONGO_URI'); and comment out the line containing the hardcoded $mongoUri value.

Commit these changes to your Git repository. Render will automatically detect this commit and initiate a new deployment.

5.3. Set Environment Variable on Render

This step involves securely providing your MongoDB connection string to your deployed application without exposing it in your codebase.

  1. Access your Render Dashboard and navigate to your PHP web service (e.g., popup).
  2. Select the "Environment" tab from the left sidebar.
  3. Click "Add Environment Variable".
  4. Define the variable:
    • Key: MONGO_URI
    • Value: Paste the full, corrected MongoDB Atlas connection string (e.g., mongodb+srv://[YOUR_DB_USERNAME]:[YOUR_DB_PASSWORD]@[YOUR_MONGODB_ATLAS_CLUSTER_HOSTNAME]/?retryWrites=true&w=majority).
  5. Click "Save Changes". Render will then initiate a redeployment.

5.4. Whitelist Render's Outbound IPs in MongoDB Atlas

This is a critical security measure, ensuring only Render's servers are authorized to connect to your MongoDB Atlas database.

  1. Return to your Render service dashboard.
  2. Click the "Connect" button (located in the top-right corner).
  3. Navigate to the "Outbound" tab.
  4. Copy all IP addresses listed.
  5. Proceed to your MongoDB Atlas dashboard: "Security" > "Network Access".
  6. Click "Add IP Address".
  7. For each IP address copied from Render, paste it into the IP Address field and append /32 (e.g., X.X.X.X/32). Add a descriptive label like "Render Outbound".
  8. Click "Confirm" or "Add Entry" for each added IP.
  9. Click "Finish and Close".

5.5. Monitor Deployment and Test

Observe the deployment progress in Render's "Events" or "Logs" tab. Once the deployment status indicates "Live", access your Render application's URL (e.g., https://your-service-name.onrender.com/) to verify the MongoDB connection and application functionality in the live environment.

6. Troubleshooting Common Issues

This section outlines common issues and their respective solutions:

  • PHP Fatal error: Class 'MongoDB\Client' not found:
    • Ensure composer install has been executed locally.
    • Verify that the path in require __DIR__ . '/../vendor/autoload.php'; within config.php correctly points to Composer's autoloader.
    • Confirm php_mongodb.dll is correctly installed and enabled in php.ini (check phpinfo() for "mongodb" section).
  • MongoDB Connection failed: Failed to look up SRV record...:
    • If encountered locally: Temporarily disable any active VPN.
    • If encountered locally: Configure your computer's DNS servers to use reliable public DNS (e.g., Google DNS: 8.8.8.8, 8.8.4.4 or Cloudflare: 1.1.1.1, 1.0.0.1).
    • If encountered locally: Flush DNS cache (ipconfig /flushdns) and restart your PC/server.
  • TLS handshake failed: error:0A000438:SSL routines::tlsv1 alert internal error...:
    • Most common: Ensure your current public IP address is accurately whitelisted in MongoDB Atlas Network Access. Dynamic IPs may change.
    • Temporarily disable any **third-party antivirus/internet security software's web protection or firewall features** (beyond Windows Defender, if applicable). Re-enable after testing.
    • Confirm that your computer's system time is accurately synchronized.
    • Verify php_openssl.dll is enabled in php.ini.
  • VS Code displays red underlines on MongoDB classes:
    • Save all modified files (Ctrl+S).
    • Reload the VS Code window (via Command Palette: Ctrl+Shift+P then "Reload Window").
    • Fully close and reopen VS Code. This is typically a linter display anomaly and does not affect code execution.

7. Admin Settings Details

The administrative panel provides comprehensive controls for the popup modal system:

  • Enable/Disable: Toggles the global visibility of the popup.
  • Trigger: Configures how the popup is activated (e.g., exit intent, time delay).
  • Target Page: Specifies whether the popup appears on all pages or a defined specific page.
  • Image, Text, CTA: Allows full customization of the popup's content, including images, main text, and Call-to-Action elements.
  • Cookie Duration: Controls the frequency at which the popup reappears for returning users.
  • Note on Mobile Triggers: For mobile devices, currently only the 'delay in seconds' option is available as a trigger.

8. Frequently Asked Questions (FAQ)

  • Q: Can this plugin be integrated with any PHP site?
    A: Yes, this plugin is developed in pure PHP and does not require any specific framework, making it highly portable.
  • Q: How is the popup triggered?
    A: Popups can be triggered by exit intent, a configurable time delay, or based on scroll percentage. For mobile devices, the only currently supported trigger is a configurable time delay in seconds.
  • Q: Where are uploaded images stored?
    A: Uploaded images for popup content are stored in the /uploads/images/popup/ directory within the project structure.
